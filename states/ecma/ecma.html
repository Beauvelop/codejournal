Very simple example of the prototype-chain

  

var taxi = {
  kind: "taxi",
  color: "yellow",
  features: "legroom",
  hornType: "loud"
};

taxi.wheels = 4;

Object.prototype.speed = function() {
  console.log("~ZZZZoooooom!");
};

var limo = Object.create(taxi);
limo.kind = "limo";

// Question: What happens if I call a method not placed directly on 'limo'?

limo.speed();

// "~!ZZZZoooooom!"



Explanation:


1) JS doesn't find .speed() on "limo" --doesn't find it.
2) It then looks at "taxi" --doesn't find it.
3) Finally, javascript finds it on the root prototype: "Object.prototype"... finds it!



In this way, limo inherits the .speed() method from the root prototype Object.prototype

Let's use this in the speed method to see what happens.


Object.prototype.speed = function() {
  console.log("The " + this.kind + " goes '~ZZZZoooooom!'");
};

// Now calling...
limo.speed();
taxi.speed();

// "The limo goes '~!ZZZZoooooom!'"
// "The taxi goes '~!ZZZZoooooom!'"





Let's test these.

I. We can reason thattaxiis built on Object.prototype
II. We know further that limois built on the prototype oftaxi

We can test them withObject.getPrototypeOf(variable)


var taxiProto = Object.getPrototypeOf(taxi);
var limoProto = Object.getPrototypeOf(limo);

console.log(limoProto === taxi);  // true!
console.log(taxiProto === Object.prototype); // true!


QED!üéâ

Here we have proof of the above assertions about the prototype chain.

function makeGreeting(language) {

  var counter = 0;
  return function(firstName, lastName) {
    counter++;
    var lang;
    if(language === "es") {

Capturing vs Bubbling

  
Propagation is about how events are registered and 'caught' vis-a-vis other elements. Here, e.stopPropagation() is called to prevent any events on parent elements. Therefore, we don't log to console


  
2 ways to register an event.
You can 'catch' the event in the Capturing Phase so that a 'higher' DOM element listens down to its children for an event. div->ul->li->a 


Otherwise, an evt can be registered on the lowest element until it bubbles up to the div 

This is called capturing the event in the Bubbling Phase

  
  document.getElementById('container').addEventListener('click', function(e) {

  When you add an event handler to the event target itself, bubble vs capture is not applicable.
  At that level, you're not capturing or bubbling--you're on the target itself.
  The event handlers fire in the order in which they were registered.
