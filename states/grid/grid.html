CSS GRID: A walkthrough

Line-based layout in GRID

Line based layouts describe elements according to the lines that delineate them. --not with "width" and "height"


So, grid-column: 1/3; => "starts at line #1, ends at line #3"
-1 = last line
-1 is frequently used to tell the element to just "start where you are and go to the end!"

Drying out Grid

If your element only spans one track, the slash + num is superfluous.

"45. grid-column: 3;"

Notice that the "coral" div has a "hole" beside it. That is because I only told it to span one track!


The "span" keyword:

Notice that if you want the element to span a number of tracks, just put span + number

"53. grid-column: 1/span 3;"



NOTE: grid-row: and grid-column: are, in fact, short-hand for "grid-column-start:", "grid-column-end:" etc.


But why bother with these? --You already know the cool stuff!

.grid {
 display: grid;

 grid-template-columns: repeat(3, 1fr);

 grid-template-rows: minmax(100px, auto);

 grid-gap: 10px;

 }

This .grid rule defines default height as 100px unless content overrides this value.
Then, the "auto" will enable a stretch to accomodate content.

fr = "fraction".
It accounts for fixed and relative gaps. "fr" respects the absolute value of px widths. These will never change! Only the fr columns will respond to viewport adjustment--which is extremely cool and one of the many reasons why so many lovely front-end nerds go on about it! ðŸ˜€
Let's dry out this css.


Imagine that we have four columns:


 grid-template-columns: 2fr 2fr 2fr 2fr;

        
Lets clean this repeated pattern up by using:Grid's repeat(); method!

        
 grid-template-columns: repeat(4, 2fr);

        
This results in four even columns.


      
Take look at this powerful style-rule...

      
grid-template-columns: repeat(2, 2fr 3fr);

That's right. We'll get 2 pairs. First column equalling 2fr the second, 3fr--and then another pair!
