Multiline comments in SASS:


/* This kind of multiline comment
   with indentation will compile to
   your css file and does not need
   to be "closed" with a "*/".

HOWEVER, if you don't wish that to be compiled to .css (but only visible in your sass file)

// simply replace the second "*" with a "/"
   You can indent all day long and so long
   as the comments begin with a double-slash you will
   never see anything in your css file.

Using calc() with css variables

You can employ calc and var() as a time time a delay with the variable declared on :root Here, it is used as a multiplicand on animation-delay 
:root {
  --tick: .2s;
}
...
#dot1 {
  animation-delay: calc(var(--tick)*2);
}
#dot2 {
  animation-delay: calc(var(--tick)*4);
}
#dot3 {
  animation-delay: calc(var(--tick)*6);
}
...
It's extremely effective and efficient. I haven't used it in sass, but I see no reason that it wouldn't work.

Centering with Translate:

This is really the best example.



.ctr-container-ex3 {
  position: relative;
  height: 40vh;
  width: 100%;
}
.ctr-example3 {
  height: 200px;
  width: 200px;
  border: 1px solid fuchsia;
  position: absolute;
  left: 50%;
  top: 50%;
  transform: translate(-50%, -50%);
}

This is centered both horiz & vertically!
